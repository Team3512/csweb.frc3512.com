What is Git?

  • Version Control System (VCS)
    • Records changes to files
    • Provides a backup plan
  • Used by major companies and more

Types of VCSs: Centralized

  • Changes are pushed to and checked out from central server
  • Examples: SVN, CVS

Types of VCSs: Distributed

  • Computers are peers, so every machine has full copy of history
  • Work can also be done offline
  • Examples: Git, Mercurial

How Git Stores Data

  • Snapshots instead of diffs
  • Git stores changed file as compressed object in Git metadata

Cloning a Repository

git clone git://github.com/schacon/grit.git
git clone git://github.com/schacon/grit.git mygrit
  • Three protocols
    • git://
    • ssh://
    • https://
  • First arg is repo
  • Second arg is optional folder name
  • git:// for anon read-only
  • ssh:// for read/write secured with SSH keys (may or may not have passphrase)
  • https:// for read/write

Exchanging Changes with Others

  • "Push" changes out
    git push [<local branch>]
    ex: git push origin master
  • "Pull" changes in (fetch + merge)
    git pull [<remote branch>]
    ex: git pull origin/master
  • "origin" is a remote
