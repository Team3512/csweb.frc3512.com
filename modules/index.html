Software Curriculum

Introduction

"Engineer based on requirements, not an ideology."

This page serves as a repository of knowledge for topics the software team has explored in the past and topics alumni are learning in college. Students probably won't be able to learn everything in one year, but students are encouraged to learn outside of build sessions to become effective members as soon as possible.

Modules

Computer Science 101 (Basics of C++)

Introduces the fundamentals of computer programming and software design. Topics include variables, data types, assignment, expressions, basic I/O, control flow, functions and parameters, scope, and data structures.

Computer Science 102 (Advanced C++)

Covers more advanced topics such as threading, synchronization, and concurrency, parallelism, asynchronous data passing (i.e. networking). Proper use of C++11 threads, mutexes, and atomics will be taught. SFML is used to instruct students in the proper use of UDP and TCP sockets, as well as when to use each type.

Computer Science 103 (Shell Scripting)

An introductory module in writing shell scripts using Bash. Shell scripts will be used to automate sequences of commands to optimize parts of one's workflow.

Computer Science 104 (HTML)

Introduces website design using HTML, CSS and Javascript. Emphasis is placed on writing HTML5 conformant web pages.

Computer Science 105 (Vision Processing)

Teaches basic vision processing using the OpenCV computer vision library.

Computer Science 106 (Perceptrons and Artificial Neural Networks)

Introduces the field of machine learning with the perceptron. A basic example in vision processing is presented.

Computer Science 107 (Android Development)

Introduces writing applications using the Android SDK. Instruction in the Java programming language is provided as needed.

Computer Science 108 (Graphical User Interfaces)

Introduces designing graphical user interfaces with the Qt graphics toolkit.

Git 101 (Basics of Git)

Introduces version control systems and their uses. This module is taught using Git. Installation of Git, setting up user identification for commits, and basic workflow are covered, with emphasis on the use of branches to manage work between repositories.

Git 102 (Gerrit)

Expands on the set of commands and techniques learned from the previous module. Topics include tags, rebasing, and managing repository history. These tools will be applied to working with Gerrit, a code review system.

Real-Time 101 (Intro to Real-Time Software)

Covers real-time requirements (e.g., predictability) and methods for meeting them.

Control Theory 101 (State Machines and PID Controllers)

Introduces the topic of control systems and how they can improve the performance of a robot. State machines and PID controllers are presented as simple yet effective tools for most control applications in FRC. The dynamics of PID controllers, their types, and procedures for tuning them effectively are also discussed.

Control Theory 102 (Motion Profiling)

This is a continuation of Control Theory 101. PID controllers are used in conjunction with motion profiles of several types to produce more consistent system responses. The implementation of motion profiles as well as tuning PID controllers which don't undergo step responses are covered.

Control Theory 103 (Classical Control Theory)

Provides an introduction to methods of classical control theory. Frequency domain analysis will be used to characterize systems using their step response. Single input, single output (SISO) systems will be discussed.

Control Theory 104 (Modern Control Theory)

Provides an introduction to methods of modern control theory. Design, analysis, and control of multiple input, multiple output (MIMO) systems will be performed using their state space representations.

Signal Processing 101 (Filters and State Estimation)

Discusses filters, why they are necessary, and their application to control theory. Several basic filters are covered such as the rolling average filter, complimentary filter, low-pass filter, and high-pass filter. Each one is studied for its respective use in predicting the state of a system.

Signal Processing 102 (Kalman Filter and Linear Algebra)

The Kalman filter's inner workings as well as its utility in state estimation will be discussed. The implementation of a single-input Kalman filter will be studied to reinforce some concepts and show an implementation for real robot code. Topics include noise distribution and some linear algebra.

FRC 101 (Robot Software)

Applies the various skills learned in RT 101, CS 101, and CT 101 to the control of electromechanical systems in FRC. Robot software design patterns are covered as well as features of WPILib and FRC team 3512's custom classes.

FRC 102 (Scouting Application)

Focuses on writing a software application for scouting during FRC competitions. This has typically been written in conjunction with a spreadsheet which receives the scouting data. HTML web pages and Android applications have been used in the past.

Submission

Assignment submissions will be made via Git. To start, make an account on GitHub if you don't already have one. Solutions for each module's assignments and labs should be within a repository named like cs101-assignments where the part before the hyphen is the module name. Use git clone to clone the repository to your local machine. git add, git commit, and git push should be used to make snapshots of your work and push it to the remote server. Make commits to your repository as often as you feel is necessary. See this cheat sheet for an overview of the most commonly used Git commands.

The directory structure should be set up as follows. The given names are examples demonstrating the desired format. Your source files may have different names from those shown or not exist. Folders use lowercase and files use camel case. Some problems or labs may require multiple files.

  • cs101-assignments (Git repository root)
    • chapter01 (workspace)
      • drill02 (project)
        • src
          • ExampleClass.cpp
          • ExampleClass.hpp
          • Main.cpp
        • Makefile
      • exercise13 (project)
        • src
          • Main.cpp
        • README.txt
    • chapter11 (workspace)
      • drill14 (project)
        • src
          • Main.cpp
        • license.txt
    • cool-lab (workspace)
      • first-section-name (project)
        • src
          • Main.cpp
          • SupportFile.cpp
          • SupportFile.hpp

Source code goes in the src directory while other files like licenses, makefiles, READMEs and data go outside of it. It's convention to define main() in a file called Main.cpp, but it isn't required and may not make sense for projects with only one file.

Labs may include more specific instructions on directory structure and which files to submit. If there are none, submit all files used and follow the aformentioned format.

Submissions will have our formatter run on them. After setting it up according to these instructions, place .clang-format, .styleguide, and .styleguide-license in the root directory of your repository. You may modify .styleguide-license to use your name.

Option 1: Eclipse Workspaces

The directory tree above provides an example of which folders should be an Eclipse workspace or project within a workspace (parentheses are not part of the name). This ensures the Eclipse projects and their metadata are backed up via Git. If you do this, you'll need to include the following entries in a .gitignore in the root directory of your Git repository: 

*.d
*.o
.metadata/
.settings/
Debug/
Release/
RemoteSystemsTempFiles/

If you need to clone your repository to a different computer to work projects in what was an existing chapter workspace, choose the chapter folder as a workspace when Eclipse starts up like usual. You can import the problem projects into the workspace by right-clicking on the Project Explorer and selecting "Import...". In the resulting dialog, choose "General > Existing Projects into Workspace".
