Eclipse Installation

Linux

Install JDK 8 via the machine's package manager. Install Eclipse CDT in the same way only if your distribution does not provide the latest version of Eclipse. Otherwise, follow the instructions below.

Windows

On the Eclipse CDT download page, select the appropriate platform on the page's right. If your machine is 64 bit, choose a 64 bit version. Once the download is complete, extract it to a directory of your choice.

On the JDK download page, select the JDK installer for the same operating system and architecture of Eclipse CDT, download it, and run it.

Eclipse Usage

When creating a new project, don't use "Cross GCC". On Linux, use "Linux GCC" and on Windows, use "Cygwin GCC".

Eclipse occasionally shows outdated or false errors in a project file. If this is the case, the project's index needs to be rebuilt. To do so, select the following option.

Right-click project > Index > Rebuild

Keep in mind errors shown in the IDE aren't authoritative. The console build output should be used to determine whether the executable was built successfully.

To configure a project to build with C++14, select "ISOC++1y" from the "Language standard" drop-down menu in the following location.

Right-click project > Properties > C/C++ Build > Settings > GCC C++ Compiler > Dialect

GCC Compiler

This is the compiler Eclipse CDT will use to build the programs you write. MSYS2 provides a Linux-like environment with this compiler.

GCC Downloads

For Windows, GCC is included in MSYS2: http://www.msys2.org/.

GCC Installation

Linux

Install gcc via the machine's package manager.

Windows

Follow the instructions on the web page for MSYS2. Use the default installation directory.

When installation is complete and everything has been upgraded with pacman, add "C:\msys64\usr\bin" or "C:\msys32\usr\bin" to your system-wide PATH environment variable depending on whether you used the x86_64 installer or i686 installer respectively.

You may want to create a desktop shortcut for "MSYS2 MSYS" to facilitate opening MSYS2 terminals. Open an MSYS2 terminal and run the following command to install make, gcc, and other required tools.

pacman -S base-devel gcc

When prompted for which packages in the base-devel group to install, press Enter to select the default option (all of them).

Formatter

We use the Python 3 scripts provided in WPILib's style guide repository and clang-format for source code formatting. We also use Python 3 for optimizing feedback control system behavior.

Formatter Downloads

LLVM

Formatter Installation

On Linux, run the following commands to install the packages relevant to your platform instead of following the Windows instructions below. The clang-format version installed should be 3.8.1 or newer.

*buntu 

sudo apt-get install clang-format python3-pip
pip3 install --user wpiformat

Arch Linux 

sudo pacman -S clang python python-pip
pip install --user wpiformat

Windows

On the LLVM download page, select the installer for latest version of LLVM corresponding to your operating system and architecture under "Pre-Built Binaries", then run it. The default installation options are fine.

MSYS2 keeps a separate PATH environment variable from the operating system. Run the following command to add the LLVM installation to it.

echo 'export PATH=$PATH:/c/Program\ Files/LLVM/bin' >> .bashrc

The following commands install Python and pip, then use pip to install wpiformat. 

pacman -S python
wget https://bootstrap.pypa.io/get-pip.py
python get-pip.py
pip install wpiformat

After those commands complete, close and reopen the terminal so the PATH environment variable change from earlier takes effect.

Formatter Usage

If the --user flag was supplied to pip when installing wpiformat, run python3 -m wpiformat in the Git repository being formatted. Otherwise, run wpiformat. Therefore, if the installation instructions above were followed, use python3 -m wpiformat on Linux and wpiformat on Windows.
