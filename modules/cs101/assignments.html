Assignment Submission

Assignment submissions will be made via Git. To start, make an account on GitHub if you don't already have one, and create a new repository called cs101-assignments. Use git clone to clone the repository to your local machine. git add, git commit, and git push should be used to make snapshots of your work and push it to the remote server. Make commits to your repository as often as you feel is necessary. See this cheat sheet for an overview of the most commonly used Git commands.

The directory structure should be set up as follows. The given names are examples demonstrating the desired format. Your source files may have different names from those shown or not exist. Some problems will require multiple files, but problems from the earlier chapters typically do not.

  • cs101-assignments (Git repository root)
    • Chapter01 (workspace)
      • Drill02 (project)
        • src
          • ExampleClass.cpp
          • ExampleClass.hpp
          • Main.cpp
        • Makefile
      • Exercise13 (project)
        • src
          • Main.cpp
        • README.txt
    • Chapter11 (workspace)
      • Drill14 (project)
        • src
          • Main.cpp
        • license.txt

Source code goes in the src directory while other files like licenses, makefiles, and READMEs go outside of it. It's convention to define main() in a file called Main.cpp, but it isn't required and may not make sense for projects with only one file.

Option 1: Eclipse Workspaces

The directory tree above provides an example of which folders should be an Eclipse workspace or project within a workspace (parentheses are not part of the name). This ensures the Eclipse projects and their metadata are backed up via Git. If you do this, you'll need to include the following entries in a .gitignore in the root directory of your Git repository:

*.d
*.o
.metadata/
.settings/
Debug/
Release/
RemoteSystemsTempFiles/

If you need to clone your repository to a different computer to work projects in what was an existing chapter workspace, choose the chapter folder as a workspace when Eclipse starts up like usual. You can import the problem projects into the workspace by right-clicking on the Project Explorer and selecting "Import...". In the resulting dialog, choose "General > Existing Projects into Workspace".
