Assignment Submission

Assignment submissions will be made via Git. To start, make an account on GitHub if you don't already have one, and create a new repository called cs101-assignments. Use git clone to clone the repository to your local machine. git add, git commit, and git push should be used to make snapshots of your work and push it to the remote server. Make commits to your repository as often as you feel is necessary. See this cheat sheet for an overview of the most commonly used Git commands.

The directory structure should be set up as follows. The given names are examples demonstrating the desired format.

  • Chapter01
    • Problem02
      • src
        • ExampleClass.cpp
        • ExampleClass.hpp
        • Main.cpp
    • Problem13
      • src
        • Main.cpp
  • Chapter11
    • Problem14
      • src
        • Main.cpp

It's convention to define main() in a file called Main.cpp, but it isn't required and may not make sense for projects with only one file.

Option 1: Eclipse Workspaces

One option for submission using the above format is to make each chapter folder an Eclipse workspace and make each problem folder a project in that workspace. This ensures the Eclipse projects and their metadata are backed up via Git. If you do this, you'll need to include the following entries in a .gitignore in the root directory of your Git repository:

*.d
*.o
.settings/
Debug/
Release/
RemoteSystemsTempFiles/

Option 2: Alternate Editor

If you want to use a different IDE such as Vim, provide a Makefile in each problem folder to compile the executable. Here's an example Makefile which recursively finds all the source files in the src directory and compiles Debug and Release versions of them. The NAME field should be edited before use. The following entries should be included in the repository's .gitignore file:

*.d
*.o

Assignments

In addition to the following problems, do all the drill problems at the end of each chapter. See section 0.1.2 in the textbook for the reasoning behind this and section 0.1.1 for the textbook's general approach. The review questions are good for checking comprehension, but they aren't exhaustive, so still read carefully.

We recommend covering at least one chapter per week. There may not be enough time to lecture on all the material in depth in a given week, so individual reading is encouraged. The book is designed for this situation.
